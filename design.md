# 大数据平台 AI Agent 问题闭环工作台 - MVP 设计文档

## 1. 文档概述

### 1.1 目的
本文档旨在定义“大数据平台 AI Agent 问题闭环工作台”的最小可行性产品（MVP）的技术架构、核心功能和实现路径。MVP 的核心目标是验证利用 Claude Agent 对图文报错进行自动化诊断的可行性，并跑通基础的人机交互流程。

### 1.2 业务背景
* **前提**：大数据平台已有基于 Claude Agent 的智能客服。
* **痛点**：用户提工单（传声）数量大，系统使用和排查门槛高。
* **MVP 目标**：构建一个桌面端程序，允许客服通过图文上传问题，由后台的 AI Agent 进行初步解答，以此替代部分纯人工排查动作。

## 2. MVP 产品定义 (Scope of MVP)

MVP 将专注于最核心的诊断和人机交互环节。

### 2.1 核心用户场景
1. 客服在桌面端收到用户发来的报错截图或文字描述。
2. 客服打开工作台，上传截图和输入描述文本。
3. 点击“诊断”，问题被发送给后台运行的 Claude Agent。
4. AI Agent 返回诊断简报和建议回复。
5. 客服审核并拷贝回复发送给用户。

### 2.2 核心功能模块 (MVP 范围)
* **桌面端 UI (用户端)**
  * **主界面**：一个极简的聊天/诊断窗口。
  * **问题输入**：支持文字输入框。
  * **多媒体支持**：支持图片上传（报错截图）和文本文件上传（日志片段）。
  * **诊断触发**：明确的“开始诊断”按钮。
  * **结果展示**：展示 AI Agent 的诊断报告和建议回复（支持 Markdown 渲染）。
* **AI Agent 核心 (后端)**
  * **SDK 集成**：利用 Node.js 集成 Claude Agent SDK。
  * **多模态理解**：调用具有 Vision 能力的模型（如 Claude 3.5 Sonnet）处理图片。
  * **Prompt 工程**：设计专用 Prompt，引导 AI 识别大数据报错模式，并输出结构化的诊断简报。

### 2.3 MVP 暂不包含 (非 Scope)
* 丰声插件/IM 的直接打通。
* 平台原子能力的一键修复操作（MCP 插件调用）。
* 后台管理与运营平台。
* 复杂的知识库查重决策与自动更新。

## 3. 技术架构选型

MVP 推荐采用纯前端/Node 相关的技术栈，便于快速迭代和构建桌面端。

* **桌面端壳 (Desktop Shell)**：**Electron**。业界标准，利用 Web 技术构建跨平台桌面应用的首选方案，上手快。
* **前端 UI (Frontend)**：**React + Tailwind CSS**。React 适合处理复杂的状态流转（上传中、诊断中等），Tailwind CSS 能够极速构建原型样式。
* **Agent 通信层**：**Node.js (Electron Main Process)**。安全地处理文件读取、环境变量（API Key）以及与 Claude SDK 的网络通信。
* **开发语言**：**TypeScript**。在对接 SDK 和管理复杂状态时，提供必要的类型安全。
* **AI 模型**：**Claude 3.5 Sonnet**。兼顾强大的代码/日志推理能力和优秀的视觉（Vision）图文识别能力。

## 4. 核心实现建议

1. **进程隔离保安全**：坚决不要把 Claude API Key 放在 React 渲染进程中。所有的 SDK 调用必须在 Electron 的主进程（Main Process）中完成，前端通过 IPC (Inter-Process Communication) 发送图文数据并接收结果。
2. **多模态 Prompt 构建**：在使用 Claude SDK 时，将用户的文字描述和图片的 Base64 编码组合成结构化的 `messages` 数组发送。
3. **状态机设计**：前端需要设计一个清晰的诊断状态机：`IDLE` (空闲) -> `UPLOADING` (文件处理中) -> `DIAGNOSING` (请求大模型中) -> `SUCCESS` (返回结果) / `ERROR` (请求失败)。

## 5. 成功指标
* **技术可用性**：系统能成功读取截图报错并返回结构化解答。
* **体感效率**：客服完成单次图文预排查的时间控制在 10-15 秒内。
